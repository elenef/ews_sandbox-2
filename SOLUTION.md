### Куда положили state и почему

не добавляла новый глобальный state, чтобы не дублировать URL/API cache

URL state: id выводится из URL (`/tickets/:ticketId`) в `WorkspaceShell`.
API/cache state: список и детали тикетов получаем с RTK Query запросов, обновление через мутацию
Browser state: последние ids тикетов хранятся и достаются из localStorage

### где проходит граница command palette / ticket feature

в проекте существует CommandPalette, который заводится в workspaceShell, отдельным входным аргументом собираем команды и крепим функции для run в отдельном слое workspace/commands где храним ticket-aware + глобальные команды. Не импортируем ticket API/types/business logic. Соответственно, в CommandPalette лежит вся ран тайм настройка работы паллета

### какие edge cases покрыты

в readRecentTicketIds обработаны случаи отсутствия хранилища, проблемы с парсингом, если там лежит не массив, также проверено чтобы строки были непустые, были строкой и также обрезаны trim()
Фильтрация по существующим тикетам перенесена в слой работы со списком тикетов

### какие trade-offs приняли

Не вводила глобальный стор и локальный стейт, так как ids нужны были только на одной странице без учета вкладок
